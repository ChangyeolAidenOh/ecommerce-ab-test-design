# Case 1: GIF 피드 밀도 실험 설계서

**Experiment Design Document**
**작성일:** 2026-05-04
**상태:** 설계 완료 (시뮬레이션 검증 포함)

---

## §1. Business Context & Problem Definition

### 배경

에이블리 홈 피드에서 셀러가 등록한 상품 이미지 중 일부가 GIF로 자동 재생된다. GIF는 정적 이미지 대비 시각적 주목도가 높아 상품 클릭률(CTR)을 올릴 수 있지만, 여러 GIF가 연속으로 배치될 경우 시각 피로를 유발하여 세션 이탈을 증가시킬 가능성이 있다.

### 현재 상태

에이블리 앱(v2.364.0)의 설정 > 실험실 메뉴에서 "움직이는 이미지(gif) 기능"을 유저가 직접 on/off할 수 있다. 이 구조는 **self-selection**이므로 GIF의 인과적 효과를 측정할 수 없다.

시뮬레이션 검증 결과, self-selection 환경에서의 효과 추정치는 RCT 추정치 대비 **4.0배 과대 추정**되었다 (selection bias = true effect의 132%). GIF를 끄는 유저는 원래 baseline 전환율이 낮은 집단이기 때문에, 단순 on/off 비교는 GIF의 실제 효과를 왜곡한다.

### 마켓플레이스 레버 제약

에이블리는 셀러가 이미지를 직접 등록하는 마켓플레이스 구조다. 셀러는 본인이 직접 촬영한 사진만 사용 가능하며 거래처 사진은 사용 불가하다 (에이블리 셀러스퀘어 입점 가이드). 따라서 셀러의 GIF 업로드 여부는 에이블리가 통제할 수 없다.

**에이블리가 통제 가능한 레버:** 피드 알고리즘에서 GIF 상품의 연속 노출 한도를 조절하는 것.

### 이 실험을 하지 않으면?

GIF의 실제 효과를 모른 채 "실험실" 토글의 self-selection 데이터로 의사결정을 하게 되며, 이는 과대 추정된 효과에 기반한 잘못된 결론으로 이어질 수 있다.

---

## §2. Hypothesis

**Primary Hypothesis:**

> 피드 내 GIF 연속 노출 한도를 3개로 제한하면(사이에 정적 이미지 1개 이상 삽입), GIF 무제한 배치 대비 세션당 구매 전환율이 감소하지 않을 것이다.

**기대 부수 효과:** GIF 연속 노출 제한으로 시각 피로가 줄어들어 세션 이탈률이 감소할 수 있다. 단, 이탈률 감소는 Primary가 아닌 Guardrail 수준에서 관찰한다.

### 실험의 목적

이 실험은 "GIF를 줄이면 더 좋아지는가?"가 아니라 **"GIF를 줄여도 괜찮은가?"**를 검증한다. GIF 밀도 제한의 1차 목적은 유저 시각 피로 감소이며, 그 전제 조건으로 "밀도를 줄여도 매출(CVR)이 떨어지지 않는다"를 확인해야 한다. 따라서 Primary metric은 이탈률이 아닌 CVR이며, 이탈률은 Guardrail로 모니터링한다.

이 가설은 "GIF를 완전히 끄자"가 아니라 **"최적의 밀도를 찾자"**라는 프레이밍이다. 에이블리가 실제로 실행할 수 있는 레버(피드 알고리즘의 연속 노출 한도)에 정확히 맞춘 가설이다.

---

## §3. Metric Hierarchy

| 구분 | 지표 | 정의 | 선택 근거 |
|---|---|---|---|
| **Primary** | 세션당 구매 전환율 (Session CVR) | 구매 완료 세션 / 전체 세션 | 매출과 직결되는 최종 전환 지표. 이 실험의 성공/실패를 판정하는 단일 기준. |
| Secondary | 세션당 상품 클릭 수 | 세션 내 상품 상세 진입 횟수 | GIF의 시각적 주목 효과를 측정. Primary와 방향이 같을 수도, 다를 수도 있음(GIF가 클릭은 늘리지만 전환은 못 올리는 경우). |
| Secondary | 세션 체류시간 | 세션 시작 ~ 마지막 이벤트 시간 | 탐색 몰입도의 프록시. 단, 길다고 반드시 좋은 건 아님 (의사결정 지연, 방황 가능성). |
| **Guardrail** | 세션 이탈률 (Bounce rate) | 상품 클릭 없이 이탈한 세션 비율 | GIF 밀도 변경이 피드 첫인상을 해치지 않는지 확인. NIM=+1.0%p. |
| **Guardrail** | 피드 스크롤 깊이 | 세션당 스크롤한 상품 카드 수 | GIF 제한이 탐색 동기를 꺾지 않는지 확인. NIM=-5%. |

### 지표 선택 시 고려하지 않은 대안과 이유

- **매출액**: 전환율의 후행 지표이면서 상품 가격에 의한 분산이 커서 실험 기간 내 통계적 유의성 확보가 어려움.
- **앱 삭제율**: GIF 밀도 변경만으로 앱 삭제까지 가는 경로가 너무 길어 노이즈가 지배적.
- **셀러 GIF 업로드율**: 이 실험은 유저 경험에 대한 것이므로 셀러 지표는 별도 분석 대상.

## §4. Experiment Design

### Randomization Unit: 유저

세션 단위로 배정하면 같은 유저가 두 variant를 번갈아 경험하여 학습 효과(carryover)가 발생한다. 유저 단위로 고정하면 일관된 경험을 제공하면서 carryover를 방지할 수 있다.

### Target Population

전체 에이블리 앱 활성 유저. GIF 상품이 피드에 존재하는 한 모든 유저에게 적용 가능.

단, 기존 "실험실"에서 GIF를 직접 off한 유저는 이미 GIF에 대한 부정적 선호를 표현한 집단이므로, 실험 대상에서 **제외하거나 별도 층화 분석**을 수행한다.

### Variants

| Variant | 설명 |
|---|---|
| Control | 현행 — GIF 연속 노출 제한 없음 |
| Treatment A | GIF 연속 3개 한도 — 3개 연속 노출 후 정적 이미지 1개 이상 삽입 |
| Treatment B | GIF 연속 2개 한도 |

3개 arm을 선택한 이유: on/off 이분법이 아니라 "최적 밀도 구간"을 탐색하기 위함. 2개 한도(B)가 3개 한도(A)보다 더 효과적인지를 동시에 비교할 수 있다.

### Sample Size & Duration

시뮬레이션 기반 power analysis 결과 (baseline CVR=3.2%, MDE=0.3%p, alpha=0.05, power=0.80):

| 항목 | 값 |
|---|---|
| 필요 표본 (보정 전) | 그룹당 56,472명 |
| Bonferroni 보정 alpha | 0.025 (3-arm, 2개 비교) |
| 필요 표본 (보정 후) | 그룹당 68,388명 |
| DAU | 50,000 [추정] |
| 트래픽 배정 | 50% |
| 추정 소요 기간 | **9일** |

2주 실험 시 탐지 가능한 최소 효과(MDE)는 0.23%p로, 설정한 0.3%p보다 작으므로 2주면 충분한 검정력을 확보한다.

### MDE 설정 근거

0.3%p를 MDE로 설정한 비즈니스 근거: baseline CVR 3.2%에서 0.3%p는 약 9.4%의 상대적 변화에 해당한다. 이 이하의 효과는 피드 알고리즘 변경의 구현·운영 비용 대비 비즈니스 임팩트가 미미하다고 판단.

### Ramp-up Plan

| 단계 | 트래픽 배정 | 기간 | 목적 |
|---|---|---|---|
| 1 | 1% | 1일 | 기술적 버그 및 데이터 파이프라인 점검 |
| 2 | 10% | 3일 | 가드레일 초기 확인 (bounce rate, scroll depth) |
| 3 | 50% | 본실험 기간 | 통계적 유의성 확보 |

## §5. Decision Framework

실험 결과가 나오기 **전에** 합의하는 의사결정 매트릭스:

| Primary (Session CVR) | Guardrail (Bounce, Scroll) | 결정 |
|---|---|---|
| Treatment ≥ Control (유의) | 비열등 (NIM 이내) | **출시** |
| Treatment ≥ Control (유의) | 열등 (NIM 초과) | **범위 축소 후 재실험** — GIF 한도를 완화하거나 타겟 유저 제한 |
| Treatment ≈ Control (비유의) | 비열등 | **폐기** — 효과 없이 복잡성만 추가 |
| Treatment < Control (유의) | — | **즉시 중단** |

### Guardrail 판정: 비열등성 검정(NIT)을 선택한 이유

열등성 검정(IT)은 "악화가 없다"를 증명하는 것이 불가능하다 — 귀무가설 기각 실패가 "동등"을 의미하지 않기 때문이다. NIT는 "NIM 이내의 악화는 비즈니스적으로 허용한다"는 실용적 판단이 가능하다.

시뮬레이션 검증 결과, bounce rate가 +0.5%p 악화된 상황에서 NIM=1.0%p로 설정하면 NON-INFERIOR 판정(출시 가능), NIM=0.8%p로 설정하면 INFERIOR 판정(출시 불가)으로 결론이 뒤집어졌다. Critical NIM(판정이 바뀌는 경계)은 0.84%p였다. NIM=1.0%p는 이 경계보다 충분히 여유 있는 설정이므로 합리적이다.

이 분석은 에이블리 DA 블로그의 "가드레일 지표의 열등성 검정 → 비열등성 검정 전환" 논의와 직접 대응한다.

## §6. Methodology Selection

### Primary Metric 검정

3개 arm 실험이므로 **다중 비교 보정**이 필요하다.

- 1차: Benjamini-Hochberg(BH)로 FDR 통제
- Bonferroni와의 결론 차이를 병기하여 보수적 판단의 근거 제공

Primary metric(Session CVR)에 대해 **Frequentist z-test**를 메인으로 사용하되, **Bayesian AB test**(Beta-Binomial conjugate)를 보조로 병행한다.

| 방법론 | 판정 기준 | 장점 | 한계 |
|---|---|---|---|
| Frequentist | p < 0.05 (BH 보정) | 업계 표준, 해석 명확 | 고정 표본 필요, peeking 취약 |
| Bayesian | P(B>A) > 0.95 | 작은 표본에서도 가능, 기대 손실 제공 | 사전 분포 설정 민감 |

시뮬레이션에서 두 방법론의 결론이 일치했으나, **effect 0.25%p 부근에서 결론이 갈렸다** — Frequentist는 p=0.079로 비유의, Bayesian은 P(B>A)=0.96으로 유의. 이 경계 조건이 발생하면 Bayesian의 기대 손실(expected loss)을 추가 판단 기준으로 사용한다.

### Peeking 위험

실험 기간 중 매일 p-value를 확인하면 Type I error가 팽창한다. 시뮬레이션 결과, **500명 간격으로 peeking 시 FP rate가 5% → 25.6%로 5.1배 팽창**했다.

따라서 중간 확인은 **주 1회**로 제한하고, 최종 판정은 사전 정의 기간 종료 후에만 수행한다. 가드레일 모니터링만 별도로 더 자주 확인할 수 있으나, Primary metric 판정에는 사용하지 않는다.

## §7. Risks & Limitations

| 위험 요인 | 영향 | 대응 |
|---|---|---|
| 노벨티 효과 | 피드 구성 변경에 대한 초기 호기심으로 효과가 과대 추정될 수 있음 | 실험 첫 3일 데이터를 제외한 분석을 병행 |
| 셀러 생태계 영향 | GIF 노출 제한 시 GIF 셀러의 노출 기회가 줄어 반발 가능 | 이 실험에서는 유저 지표만 측정. 셀러 임팩트는 별도 분석 필요 |
| 계절성 | 패션 시즌에 따라 전환율 baseline이 변동 | 실험/대조군 동시 진행이므로 상쇄. 단, 시즌 전환기 시작은 피할 것 |
| self-selection 교란 | 기존 "실험실" 토글 유저가 실험 대상에 포함되면 교란 | 실험실에서 GIF를 직접 off한 유저를 실험 대상에서 제외 또는 층화 분석 |
| DAU 추정 불확실 | DAU가 50,000이 아니라 30,000이면 실험 기간이 9일 → 14일로 늘어남 | DAU ±50% 범위에서 기간 민감도 분석 수행 |
| 가드레일 초기 미확인 | 표본이 적을 때 CI가 넓어 비열등성을 확인할 수 없음. 시뮬레이션에서 n < 20,000에서 UNCONFIRMED 상태 관찰 (INFERIOR가 아닌, 표본 부족으로 인한 판단 보류). n ≥ 20,000에서 SAFE로 전환 | 가드레일 판정은 n ≥ 20,000 이후부터 시작. 초기 UNCONFIRMED 구간에서는 INFERIOR(point estimate가 NIM 초과) 여부만 모니터링하여 조기 중단 판단 |

### 비즈니스 임팩트 환산

실험이 성공할 경우의 기대 효과:
- DAU 50,000 × baseline CVR 3.2% = 일 1,600건 전환
- MDE +0.3%p 달성 시: 일 +150건 추가 전환
- 연간 환산: 약 54,750건 추가 전환
- 에이블리 평균 객단가를 2만원으로 추정하면: 연간 약 10.9억원의 추가 매출 기대

단, 이 실험의 1차 목적은 "CVR이 떨어지지 않음을 확인"이므로, 위 환산은 CVR이 오히려 개선될 경우의 upside 시나리오다. CVR이 동등하게 유지되면서 이탈률만 개선되는 것도 성공으로 판정한다.
