# Case 2: 체형 리뷰 정렬 실험 설계서

**Experiment Design Document**
**작성일:** 2026-05-04
**상태:** 설계 완료 (시뮬레이션 검증 포함)

---

## §1. Business Context & Problem Definition

### 배경

에이블리에는 두 레벨의 정렬이 존재한다.

**상품 목록 레벨** — 카테고리 탐색 시 적용. 옵션: 추천순, 인기순, 최신순, 리뷰 많은순, 낮은 가격순, 높은 가격순. "리뷰 많은순"은 **전체 리뷰 수** 기준이다.

**리뷰 내 레벨** — 상품 상세의 리뷰 탭에서 적용. 키(5cm 버켓), 상의 사이즈(한국 의류 사이즈 44~99+), 체중(5kg 버켓), 옵션, 만족도로 필터링 가능. "내 체형만" 체크박스로 마이페이지에 등록한 체형과 일치하는 리뷰만 표시.

### 문제

상품 목록 레벨의 "리뷰 많은순" 정렬이 체형 필터 활성화 상태를 반영하지 않는다. 체형 필터를 켠 유저에게도 전체 리뷰 수 기준으로 상품이 정렬되므로, **"내 체형 리뷰가 풍부한 상품"이 상위에 노출되지 않을 수 있다**.

### 마켓플레이스 레버 제약

셀러 착용샷 보정이 과도하다는 유저 불만이 커뮤니티에서 반복적으로 관찰된다. 에이블리는 셀러 이미지를 직접 수정할 수 없으므로(셀러가 본인 촬영 이미지만 등록 가능), 유저의 구매 판단을 돕는 데 **체형 리뷰의 역할이 더욱 중요**하다. 체형 리뷰 접근성을 높이는 것은 에이블리가 통제 가능한 레버 내에서 핏 만족도를 개선하는 방법이다.

### 이 실험을 하지 않으면?

체형 필터를 적극적으로 사용하는 유저가 정작 상품 탐색 단계에서는 체형과 무관한 정렬을 경험하게 된다. 체형 리뷰 기능의 가치가 상품 상세 페이지에만 한정되어 상품 발견(discovery) 단계까지 확장되지 못한다.

---

## §2. Hypothesis

**Primary Hypothesis:**

> 체형 필터 활성화 유저에게 상품 목록 정렬 기본값을 '내 체형 리뷰 많은 순'으로 변경하면, 현행 '전체 리뷰 많은 순' 대비 **상품 상세 → 구매 전환율이 0.5%p 이상 증가**할 것이다.

**Secondary Hypothesis:**

> 동일 조건에서 **구매 후 반품률이 1.0%p 이상 감소**할 것이다.

반품률 가설의 논리: 체형 리뷰를 통해 "나와 비슷한 체형의 사람이 이 옷을 어떻게 느꼈는지"를 사전에 파악하면 핏 불만족에 의한 반품이 줄어든다.

## §3. Metric Hierarchy

| 구분 | 지표 | 정의 | 선택 근거 |
|---|---|---|---|
| **Primary** | 구매 전환율 (CVR) | 체형 필터 on 상태에서 구매 / 상품 노출 | 정렬 변경이 구매 결정에 미치는 직접 효과. 이 실험의 성공/실패 단일 기준. |
| Secondary | 반품률 | 반품 건수 / 구매 건수 (14일 윈도우) | 핏 만족도의 후행 지표. 반품 윈도우 때문에 실험 기간이 연장됨. |
| Secondary | 상품 상세 체류시간 | 상품 상세 페이지 평균 체류시간 | 체형 리뷰를 더 꼼꼼히 열람하는지 확인. 정렬 변경의 간접 효과. |
| **Guardrail** | 상품 노출 다양성 (Gini) | 노출된 상품 ID의 Gini 계수 | 체형 리뷰가 많은 특정 상품만 반복 노출되지 않는지. NIM=+0.05. |
| **Guardrail** | 탐색 포기율 | 정렬 변경 후 즉시 이탈 또는 정렬을 수동으로 되돌린 비율 | 정렬 변경이 유저 경험을 해치지 않는지. NIM=+2.0%p. |

### "절대수의 법칙" 논점

전체 리뷰 수가 압도적으로 많은 상품은 내 체형 리뷰 수도 많을 가능성이 높다. 만약 두 정렬 기준의 순위 상관이 매우 높다면, 정렬 변경의 효과가 미미할 수 있다. 이 상관 관계를 시뮬레이션으로 사전 점검하고, 상관이 낮은 상품 구간(예: 전체 리뷰 50~200개)에서 정렬 변경의 효과가 집중되는지 subgroup analysis를 설계에 포함한다.

## §4. Experiment Design

### Randomization Unit: 유저

체형 정보는 유저에 종속되므로 유저 단위 배정이 자연스럽다.

### Target Population: 체형 필터 활성화 유저만

체형 정보를 입력하지 않은 유저에게는 체형 리뷰 정렬 자체가 불가능하다. 따라서 **마이페이지에서 체형 정보를 입력하고 "내 체형만" 필터를 1회 이상 사용한 유저**만 실험 대상이다.

설계 논점: 타겟이 전체 유저의 일부(활성화 비율 추정 15%)이므로 eligible DAU가 줄어들고, 동일 MDE를 탐지하는 데 더 긴 기간이 필요하다.

### Variants

| Variant | 설명 |
|---|---|
| Control | 현행 — "리뷰 많은 순" = 전체 리뷰 수 기준 |
| Treatment | "내 체형 리뷰 많은 순" 기본값 (유저가 수동으로 전체 리뷰 순으로 변경 가능) |

2-arm 실험. Treatment에서 유저가 수동으로 정렬을 되돌릴 수 있도록 하여 강제성을 최소화한다. 되돌린 비율 자체가 guardrail(탐색 포기율)로 측정된다.

### Sample Size & Duration

시뮬레이션 기반 power analysis 결과 (baseline CVR=4.5%, MDE=0.5%p, alpha=0.05, power=0.80):

| 항목 | 값 |
|---|---|
| 필요 표본 | 그룹당 28,408명 |
| Eligible DAU | 7,500명 (= DAU 50,000 × 체형 필터 활성화 15%) |
| 트래픽 배정 | 50% |
| 실험 기간 | **16일** |
| 반품 관찰 윈도우 | +14일 |
| **총 소요 기간** | **30일** |

4주 실험 시 탐지 가능한 최소 효과(MDE)는 0.37%p로 설정한 0.5%p보다 작으므로 4주면 검정력이 충분하다.

### MDE 설정 근거

0.5%p를 MDE로 설정한 비즈니스 근거: baseline CVR 4.5%에서 0.5%p는 약 11%의 상대적 변화에 해당한다. 체형 필터 활성화 유저는 전체 유저의 15%이므로, 이 집단에서 11%의 전환율 개선은 전체 플랫폼 매출에 대해 0.5% × 15% ≈ 0.08%p의 기여를 한다. 정렬 로직 변경의 구현 비용이 낮으므로 이 수준의 효과도 비즈니스적으로 유의미하다고 판단.

### Ramp-up Plan

Case 1과 동일한 1% → 10% → 50% 단계. 단, eligible DAU가 7,500으로 작으므로 1% 단계에서 표본이 75명/일밖에 안 돼 버그 체크 외 통계 분석은 불가능. 10% 단계(750명/일)에서 가드레일 초기 확인을 시작.

## §5. Decision Framework

| Primary (CVR) | Secondary (반품률) | Guardrail (다양성, 포기율) | 결정 |
|---|---|---|---|
| ↑ 유의 | ↓ 유의 | 비열등 | **출시** — 최선의 시나리오 |
| ↑ 유의 | 비유의 | 비열등 | **출시** — 전환율만으로 충분한 근거 |
| ↑ 유의 | any | 열등 | **재설계** — 노출 다양성 보완 로직 추가 후 재실험 |
| 비유의 | ↓ 유의 | 비열등 | **장기 실험** — 반품률 효과는 있으나 CVR 효과 불명확. 더 긴 실험 또는 MDE 재검토 |
| 비유의 | 비유의 | — | **폐기** |

### Guardrail NIM 설정 근거

- **상품 노출 다양성 (Gini):** NIM=+0.05. 체형 리뷰가 많은 인기 상품이 상위를 독점하면 롱테일 셀러의 노출이 감소한다. Gini 계수 0.05 이내의 집중도 증가는 허용하되, 이를 초과하면 플랫폼 생태계에 악영향.
- **탐색 포기율:** NIM=+2.0%p. 유저가 정렬 변경을 수동으로 되돌리는 비율이 2%p 이상 증가하면 유저 경험 훼손으로 판단.

## §6. Methodology Selection

### Primary/Secondary 판정

2-arm 실험이므로 **다중 비교 보정 불필요**.

Frequentist z-test for proportions를 메인으로 사용. 단, 체형 필터 활성화 유저만 타겟이라 sample size가 제한될 수 있으므로 (eligible DAU 7,500), Bayesian 접근을 대안으로 준비한다. Bayesian은 작은 표본에서도 "현재까지의 데이터로 어느 쪽이 나을 확률이 높은가?"를 제공할 수 있어 의사결정 보조에 유용하다.

### Guardrail 판정

비열등성 검정(NIT). Case 1과 동일한 근거.

### 반품률 분석 주의사항

반품은 구매 후 7~14일 뒤에 발생하므로 실험 기간 내 데이터가 불완전하다. 실험 종료 후 14일 관찰 기간을 명시적으로 설계에 포함하며, 실험 마지막 주 구매 건의 반품 데이터가 완성되기 전에 반품률 판정을 하지 않는다.

## §7. Risks & Limitations

| 위험 요인 | 영향 | 대응 |
|---|---|---|
| 체형 필터 활성화 비율 불확실 | 아래 민감도 분석 참조 | MDE를 보수적으로 설정(0.5%p). 부족 시 기간 연장 또는 MDE 상향 재검토 |
| 체형 리뷰 편재성 | 특정 체형 버켓(예: 160-164cm, 55-59kg)에 리뷰가 편중되면 소수 체형 유저에게는 정렬 변경이 무의미 | 체형 버켓별 subgroup analysis 설계 포함 |
| 반품 윈도우 지연 | 반품은 구매 후 7~14일 뒤에 발생하므로 실험 기간이 길어짐 | 실험 종료 후 14일 관찰 기간 명시. 반품률은 secondary이므로 CVR만으로 1차 판정 가능 |
| "절대수의 법칙" | 아래 시뮬레이션 검증 결과 참조 | 효과가 중간 순위 상품에 집중될 것으로 예상. subgroup analysis 포함 |
| 정렬 변경의 2차 효과 | 체형 리뷰가 많은 상품 = 판매량이 많은 상품일 가능성 | 정렬 변경 전후 상위 20개 상품의 overlap 비율로 사전 점검 |

### §7.1 "절대수의 법칙" 시뮬레이션 검증

1,000개 상품에 대해 전체 리뷰 수 기준 순위와 특정 체형 버켓(160-164cm, 55-59kg) 리뷰 수 기준 순위를 비교한 결과:

| 지표 | 값 | 해석 |
|---|---|---|
| Spearman rho | 0.82 | 높은 양의 상관. 전체 리뷰가 많으면 체형 리뷰도 많은 경향 |
| Top-20 overlap | 18/20 (90%) | 상위 20개 상품 중 18개는 어떤 정렬 기준이든 동일 |
| Median 체형 리뷰/상품 | 2건 | 대부분의 상품에서 특정 체형 리뷰가 극소수 |
| 인기 상품 중 체형 리뷰 0건 비율 | 2% | 전체 리뷰가 많은 상품은 대체로 체형 리뷰도 있음 |

**시사점:** 상위권(top-20)에서는 정렬 변경의 영향이 작다(90% overlap). 정렬 변경의 실질적 효과는 **중간 순위 상품(21~100위권)**에서 집중될 것으로 예상한다. 이 구간에서 "전체 리뷰는 적지만 특정 체형 리뷰가 상대적으로 풍부한 상품"이 상위로 올라오면서 해당 체형 유저의 상품 발견(discovery)이 개선되는 메커니즘이다.

또한 median 체형 리뷰가 2건이라는 점은, 버켓이 144개 조합(12×12)으로 세분화되어 있어 **대부분의 상품에서 특정 체형 리뷰가 극소수**임을 보여준다. 이는 정렬 변경의 효과가 리뷰 수가 충분한 인기 상품에서만 유의미하게 나타날 수 있음을 시사하며, subgroup analysis에서 "체형 리뷰 ≥ 3건인 상품에서만 효과를 측정"하는 필터링을 고려해야 한다.

### §7.2 체형 필터 활성화 비율 민감도

이 파라미터가 Case 2 실현 가능성을 결정하는 가장 중요한 변수다.

| 활성화 비율 | Eligible DAU | 실험 기간 | 총 기간 (+ 반품 14일) | 판단 |
|---|---|---|---|---|
| 22.5% (+50%) | 11,250 | 11일 | 25일 | 충분히 실현 가능 |
| 15% (기본) | 7,500 | 16일 | 30일 | 실현 가능 |
| 8% | 4,000 | 30일 | 44일 | 도전적이지만 가능 |
| **5%** | **2,500** | **47일** | **61일** | **실현 어려움 — MDE 상향(1.0%p) 또는 실험 재설계 필요** |
| 3% | 1,500 | 79일 | 93일 | 현실적으로 불가능 |

활성화 비율이 5% 이하라면, 이 실험은 현재 설계대로 진행하기 어렵다. 이 경우 대안으로 (1) MDE를 1.0%p로 상향하여 필요 표본을 줄이거나, (2) 체형 필터 유저뿐 아니라 "체형 정보를 등록한 전체 유저"로 타겟을 확장하거나, (3) 체형 필터 사용을 유도하는 별도 시책(예: 온보딩 프롬프트)과 결합하는 방안을 검토해야 한다.

### §7.3 버켓 크기 설계 논점 (부록)

현재 에이블리의 체형 버켓은 5cm/5kg (앱 스크린샷 확인, 총 12×12=144개 조합).

핵심 trade-off: 버켓을 좁히면 체형 유사성(precision)은 높아지지만 버켓당 리뷰 수(utility)가 줄어든다.

시뮬레이션으로 3cm/3kg, 5cm/5kg, 7cm/7kg, 10cm/10kg 별 "상품당 평균 체형 리뷰 수 ≥ 3건을 충족하는 상품 비율"을 비교하여, 5cm/5kg이 sweet spot인지 검증한다. 이 분석은 Streamlit 대시보드의 Bucket Trade-off 탭에서 인터랙티브하게 탐색 가능하다.

### 비즈니스 임팩트 환산

실험이 성공할 경우의 기대 효과:
- Eligible DAU 7,500 × baseline CVR 4.5% = 일 337건 전환
- MDE +0.5%p 달성 시: 일 +37건 추가 전환
- 연간 환산: 약 13,500건 추가 전환
- 에이블리 평균 객단가 2만원 추정 시: 연간 약 2.7억원의 추가 매출

전체 플랫폼 기여도: 체형 필터 유저는 전체의 15%이므로, 이 실험의 효과는 전체 DAU 50,000 기준으로 보면 +0.5%p × 15% = +0.075%p에 해당한다. 절대 규모는 작지만, 정렬 로직 변경의 구현 비용이 낮아 ROI는 충분히 긍정적이다.

반품률 개선 효과가 함께 나타날 경우, 반품 처리 비용(물류비, CS 비용) 절감이 추가 ROI로 기대된다.
